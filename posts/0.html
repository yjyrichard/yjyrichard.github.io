在 Java 中，排序一个数组时，Arrays.sort 方法可以接受一个自定义的比较器（Comparator）

在你提供的代码中，Arrays.sort(a, 0, cnt, (o1, o2) -> new Double(o1.y).compareTo(new Double(o2, y))); 使用了一个lambda 表达式来比较数组中的元素。

为什么要用 Double 而不是 double ？

在 Java 中，compareTo 方法是 Double 类的一个方法，用于比较两个 Double 对象的大小。而 double 是基本数据类型，没有直接的 compareTo 方法。因此，使用 Double 包装类可以方便地利用已有的 compareTo 方法进行排序。

此外，new Double(o1, y) 这样写可能是在构建一个与参考值 y 相关的比较逻辑，比如计算某种距离或差异。这种做法可能用于比较两个值与参考值 y 的接近程度，而不仅仅是比较它们的绝对大小。

总结

  • retainAll 是交集操作：它保留集合 A 中那些同时存在于集合 B 中的元素，常用于查找共同好友、关键字交集、共同感兴趣的商品等场景。

  • HashSet 的其他常用方法

    ：

    • add()：添加新元素，处理不允许重复的场景。
    • remove()：移除元素，比如从购物车中移除商品。
    • contains()：检查元素是否存在，常用于查找操作。
    • size()：获取集合大小，用于统计唯一元素数量。
    • iterator()：遍历集合中的元素。
    • addAll()：并集操作，用于合并多个集合。

HashSet 作为一个无序且不允许重复的集合，具备很高效的操作性能（例如 O(1) 的添加、删除和查找），特别适合用在需要处理不重复数据的场景中。

Priority 优先队列

PriorityQueue 是一个非常有用的数据结构，它在处理一些需要动态排序的场景时非常高效。以下是一些应用实例：

1. 任务调度

在操作系统或任务调度系统中，经常会遇到不同优先级的任务，系统会优先处理优先级高的任务。例如：

  • 操作系统中的任务调度：系统会优先调度重要或紧急的任务，并根据任务的优先级执行，使用 PriorityQueue 来动态管理和调整任务队列。
  • 打印队列：高优先级的文档可以优先打印。

2. Dijkstra算法（用于最短路径问题）

PriorityQueue 在图论中最常用于 Dijkstra 算法来寻找从某个顶点到其他所有顶点的最短路径。在此算法中，每次需要从未处理的顶点中选择具有最小距离的顶点，PriorityQueue 通过动态调整顶点的优先级（即距离）来实现高效选择。

3. Huffman编码

Huffman 编码是一种无损数据压缩算法，用于将符号编码为不等长的比特串。在构建 Huffman 树时，常需要选取当前频率最小的两个节点来合并。PriorityQueue 可以帮助每次选取最小频率的节点，进而构造最优的编码树。

4. A 搜索算法*

A* 是一种启发式搜索算法，用于解决路径规划问题，常用于导航系统和游戏开发。在算法中，PriorityQueue 用来存储搜索的节点，每次从队列中选择具有最低估值（启发值）的节点进行扩展。

5. 合并K个有序链表

假设你有 K 个有序的链表，要求将它们合并成一个有序链表。你可以使用 PriorityQueue 来存储这些链表的头部节点，并且每次弹出最小的节点，并将它的下一个节点加入队列。这样可以在 O(n log k) 的时间复杂度下完成合并。

6. 动态数据流中的中位数查找

在处理动态数据流时，可以使用两个优先队列（一个最大堆，一个最小堆）来保持数据的平衡。最大堆存储较小一半的元素，最小堆存储较大一半的元素，动态平衡两个堆，就能快速找到数据流的中位数。

7. 模拟抢票系统

在某些场景下，不同用户抢票时有不同的优先级，例如VIP用户抢票会有更高优先级。可以用 PriorityQueue 实现一个公平的抢票机制，高优先级用户的请求可以优先处理。

8. 事件驱动模拟

在事件驱动的模拟系统中，事件会根据时间顺序执行。通过使用 PriorityQueue，可以高效地处理最早的事件。每次模拟系统会从队列中取出最早发生的事件，并执行其对应的操作。
